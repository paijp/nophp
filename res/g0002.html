  • So far, we have proceeded without registering anything in the table.
  • In this case, if you access a field that is not registered in the table, it is automatically escaped and stored in a field called mixed.
  • However, this does not allow searching by SQL where or sorting by order.
  • In the above, owner and body are registered as real fields.
  • In addition, the owner is indicted (do not put any spaces or other characters in "/*indexed*/" because it is recognized internally by string processing).
  • All other items except owner and body (e.g., mail in the previous examples) will still be stored in mixed.
  • Note that if you add tables, fields, or indices in tables.php, the database will be updated by pressing the create button in the ?mode=sql screen.
    • Registered data will not be lost.
    • Add the added tables/fields/indices, if any, to the database.
    • If a field already housed in a mixed is added as a real field, the data is moved from the mixed to the real field.
